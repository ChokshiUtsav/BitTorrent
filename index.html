Hindi Framework

A text processing framework for Hindi Language

Download this project as a .zip file Download this project as a tar.gz file

Synopsis

Hindi Framework provides following functionality :

  • JSON Parsing
  • Tokenization
  • Sentence Breaker
  • StopWord Detection
  • Stemming
  • POS Tagging
  • Identifying Variations
  • Named Entity Recognition
  • Categorization

Introduction

Hindi is morphologically very rich language. It has very subtle variations in writings as well.So a generalized text processing framework would not fit for hindi. Hence, we have developed a text processing framework , specifically for hindi language with all basic functionality implemented.

Such framework is useful for building responsive UIs for hindi language and prepares ground for hindi language based applications.

Tokenization

Tokenization is the process of breaking a stream of text up into words,phrases,symbols or other meaningful elements called Tokens

For Tokenizing, we have separated the text on the basis of space so that dates, designations, abbreviations retain their original form.

Tokens :

{सूरज,की,किरणें,अब,पहाड़ों,पर,खाली,हाथ,नहीं,आएगी,बेरोजगारी,के,दर्द,के,कराहते,लोगों,के,लिए,ये,किरणें,अपने,साथ,बेगारी,का,इलाज,लेकर,आएगी}


Sentence Breaking

Sentence breaking involves seperatig sentences from paragraph of text. In Hindi we denote the end of a sentence by what is called a 'purna viraam' which is denoted by '।'. Hence, sentences sepearated based on the punctuation mark '|'

For Tokenizing, we have separated the text on the basis of space so that dates, designations, abbreviations retain their original form.


StopWord Detection

Stop Words are the most frequent occurring words in the corpus.

Our approach for identifying stop words is first finding the unigrams and maintain their counts. Unigrams has been sorted based on their counts and the top 50 unigrams are declared as stop words.

Stopword Examples :

{अपना, अपनी, अपने, अभी, अंदर, आदि, आप ,इत्यादि}


Stemming

Stemming is the term used in linguistic morphology and information retrieval to describe the process for reducing inflected (or sometimes derived) words to their word stem, base or root.

Our approach for converting a word to its stem is rule-based. This rules check whether a word ends with particular suffix from the list or not. If yes, then it removes suffix and generates stem/root of word

Word-Stem Example:

{किरणें -> किरण, बेरोजगारी -> बेरोजगार}


POS Tagging

POS(Part-Of-Speech) Tagging is process of assigning each word of sentence a tag from predefined set of POS tags.

For POS Tagging, two probabilistic models are used : Hidden Markov Model(HMM) and Conditional Random Fields(CRF).

POS Tagging Example:

{सूरज -> NN,की -> PSP ,किरणें -> NN,अब -> PRP,पहाड़ों -> NN ,पर -> PSP,खाली -> JJ,हाथ -> NN,नहीं -> NEG,आएगी -> V}


Named Entity Recognition

Named Entity Recognition is process of identifying named entities from given paragraph (group of sentences).

For NER, probabilistic model Conditional Random Fields(CRF) is implemented.

NER Example:

{सूरज -> ENAMEX_LOCATION_B, हाथ -> ENAMEX_LIVTHINGS_B}


Categorizaion

Dataset used for testing are collection of News article.It belongs to different categories like politics, entertainment, sports etc.

For identifying article's category, a model based on KNN (K nearest neighbour) as well as Naive Bayes(NB)
